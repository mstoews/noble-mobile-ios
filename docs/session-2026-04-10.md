## Architecture Notes

- All views follow the existing pattern: `@Environment(APIService.self)` for API access, `@State` for local state
- All models use `Codable` with explicit `CodingKeys` mapping `snake_case` JSON to `camelCase` Swift
- The `class` field in detail models is backtick-escaped (`` `class` ``) since it's a Swift keyword
- Base URL was changed to `http://localhost:8080/public/v1` for local development
- All new views include pull-to-refresh, loading/error/empty states, and confirmation dialogs for destructive actions
